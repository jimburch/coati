# Personal AGENTS.md (global)

Global coding preferences Codex applies across every project. Project-level
AGENTS.md files layer on top and take precedence when they conflict.

## About me

Pragmatic engineer. Values clarity over cleverness, small PRs over large,
shipping over perfecting.

## How to work with me

### Be concise

- One-sentence updates between steps; no paragraphs
- End-of-turn: 1-2 sentences. What changed + what's next.
- Skip apologies. If wrong, fix and move on.

### Bias toward action — but stop at irreversible

- Explore the codebase freely: read files, run tests, grep
- Pause before: deleting files, force-pushing, running migrations, publishing,
  sending messages, opening PRs

### Don't over-engineer

- Match existing patterns — don't refactor adjacent code opportunistically
- No speculative abstractions
- No error handling for cases that can't happen
- Comments explain WHY, not WHAT

### Commits

- Conventional Commits: `type(scope): imperative subject`
- Subject ≤ 60 chars
- Body explains WHY
- One behavior per commit
- Never amend pushed commits
- Never add Co-Authored-By or "Generated by" trailers

### PRs

- Title: Conventional Commits, ≤ 70 chars
- Body: Summary (why) → Changes (bullet list) → Test plan (checklist)
- Link the issue
- Keep under 400 lines when possible

## Stack preferences (when I'm picking)

- **Languages:** TypeScript, Go, Python, Rust
- **Frameworks:** SvelteKit, Next.js, Fastify
- **DB:** Postgres + Drizzle (or Prisma)
- **Tooling:** pnpm, Vitest, Playwright, Tailwind
- **Editor:** VS Code with Vim bindings

## Avoid

- Narrating internal reasoning
- Pleasantries ("Great question!")
- Emoji in commits, code, or docs — unless I ask
- Restating what I said
- Long end-of-turn summaries

## When in doubt

A short question beats a wrong assumption.
