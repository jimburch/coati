# CLAUDE.md — Linkly

## Project Overview

Linkly is a self-hosted link shortener with team workspaces, analytics, and
programmatic access via API keys. It is built as a single SvelteKit application
that serves both the marketing site (SSR) and the authenticated dashboard (SPA).

The app is deployed as a Node adapter build behind Caddy, with PostgreSQL
managed by the host and Redis for rate limiting.

## Tech Stack

- **Framework:** SvelteKit 2 (Svelte 5 runes)
- **Language:** TypeScript 5.6+ (strict, `noUncheckedIndexedAccess: true`)
- **Styling:** Tailwind CSS 3 + shadcn-svelte
- **Database:** PostgreSQL 16
- **ORM:** Drizzle ORM (drizzle-kit for migrations)
- **Auth:** Lucia Auth v3 + Arctic (GitHub + Google OAuth)
- **Rate limiting:** Upstash Redis via `@upstash/ratelimit`
- **Testing:** Vitest (unit) + Playwright (e2e)
- **Package manager:** pnpm (workspaces)

## Coding Conventions

- Use Svelte 5 runes (`$state`, `$derived`, `$effect`); do not write `$:` reactive statements
- Use `const` by default; never `var`
- All exported functions need explicit return types
- Use Drizzle's query builder; no raw SQL except for reporting queries in `queries/analytics.ts`
- Validate every inbound payload with Zod — never trust `request.json()` directly
- API routes return `{ data: T }` on success and `{ error: string, code: string }` on failure
- Use form actions for web mutations; use `+server.ts` endpoints only for the public JSON API
- One component per file; filename in PascalCase matches the default export
- All user-facing strings live in components, not in server files (keeps i18n easy later)

## Routing Rules

- `(marketing)/` routes are SSR — they must work without JavaScript
- `(app)/` routes are SPA — safe to use client-only features like `window`
- `api/v1/*` always returns JSON and never renders HTML
- `[slug]/+server.ts` is the hot path — keep it allocation-free where possible

## Database

- Every migration is generated via `pnpm db:generate` — never hand-write migration SQL
- New tables need an index on `workspace_id` if the workspace owns the row
- Soft deletes use a nullable `deleted_at` timestamp, not a boolean flag
- Derive types from schema with `$inferSelect` / `$inferInsert` — do not duplicate shapes

## Auth

- Every authenticated route resolves `event.locals.user` via `hooks.server.ts`
- Never read session cookies directly in a route — always trust `locals.user`
- API key auth lives in `lib/server/auth/apiKey.ts` and is mutually exclusive with cookie auth
- OAuth redirect URLs are registered with GitHub/Google — update both before shipping a domain change

## Testing

- Unit tests colocate with source: `slug.ts` → `slug.test.ts`
- E2E tests live under `e2e/` and run against a real Postgres via `docker compose up -d db`
- Integration tests for DB queries use a dedicated `linkly_test` schema that is truncated per test

## Do

- Run `pnpm check` (svelte-check + tsc) before declaring a task done
- Generate migrations any time `schema.ts` changes — forgetting this is the #1 source of prod bugs
- Keep the short-link redirect handler under 5ms p99 — it handles the entire billable surface

## Don't

- Don't add React, Next.js, or any React-flavored library
- Don't use `$:` reactive statements — we're on Svelte 5 runes
- Don't introduce another ORM; Drizzle is the single source of truth
- Don't call `fetch` from SSR routes without a timeout — a hung upstream hangs the page
- Don't commit `.env` or `.env.local`
